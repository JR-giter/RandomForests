Introduction to the package Random Forests

Alexander Kolb, Joel Reiß, Nils Blum, Luke Böhler

15.03.2026

This vignette will show you how to work with the package in the following steps:

  • Creating a decision tree for regression or classification

  • Applying cost-complexity pruning to a previously generated tree in order to reduce its size and avoid overfitting

  • Using the bagging algorithm to generate multiple bootstrap datasets and corresponding trees for improved predictions

  • Using the random forest algorithm to generate a diverse ensemble of trees to get better predictions

To illustrate the functionality, we will use the Ames Housing dataset throughout the examples.

library(RandomForests)
library(AmesHousing)
ames <- make_ames() # get dataset

Creating a decision tree for regression or classification

To generate a classification-and-regression-tree (CART) on a given dataset you use the function generate_cart_tree():

cart <- generate_cart_tree( dataSet = ames,
                            properties = 20,
                            n_nodes = 100,
                            mode = "regression",
                            target = "Sale_Price")


After that you can get the prediction for a datapoint of your choice using predict_tree() :

predict_cart(cart, datapoint)
# [1] 146000


You can also check how well your CART generally predicts the target variable by using test_cart() with existing data points from your dataset:

→ checking how well the CART performs:

test_results <- test_cart(tree = cart,
                          dataPoints = ames[500:1000,],
                          mode = "regression",
                          target = "Sale_Price")

→ the results are presented as the percentage difference between the predicted and the actual value of the target variable:

show_results(test_results)
#>       max       min      mean    median 
#> 258.16619   0.00000  24.23485  15.04425

Cost-complexyity pruning

If the CART was generated using many data points with numerous features, the function prune_tree() can be used afterwards to reduce the size:

→ number of nodes in the original tree:

count_nodes(cart)
#> [1] 199

→ cost-complexity pruning:

pruned_cart <- prune_tree(cart)

→ number of nodes in the pruned tree:

count_nodes(pruned_cart$optimal_tree)
#> [1] 45


However, this can slightly reduce the accuracy of the predictions:

test_results <- test_cart(tree = pruned_cart$optimal_tree,
                          dataPoints = ames[500:1000,],
                          mode = "regression",
                          target = "Sale_Price")

show_results(test_results)
#>          max          min         mean       median 
#> 229.89493792   0.05057284  26.03053878  15.71189280

Bagging

To further improve prediction performance, the bagging algorithm can be applied, generating multiple bootstrap datasets that are used to train several decision trees. To do this, simply use the function bagging_greedycart():

bagging_trees <- bagging_greedycart(data = ames,
                                    n_bootstrapSamples = 20,
                                    properties = 20,
                                    n_nodes = 200,
                                    mode = "regression",
                                    target = "Sale_Price")


Now you can get a prediction for your datapoint with predict_bagging():

predict_bagging(bagging_trees, datapoint, mode = "regression")
#> [1] 188948


You can also run a test here to evaluate the overall performance of the bagging model:

test_results <- test_bagging(models = bagging_trees,
                             dataPoints = ames[500:1000,],
                             mode = "regression",
                             target = "Sale_Price")


show_results(test_results)
#>          max          min         mean       median 
#> 219.32822350   0.04685315  20.92268138  14.17593540

Random Forests

Building upon the bagging approach, the Random Forest algorithm further improves prediction performance by introducing randomness in the selection of features for each split. You can use random_forest() to generate such a forest:

randomforest <- random_forest(
    data = ames[1:100, ],
    B = 50,
    m = 8,
    target = "Sale_Price",
    mode = "regression"
)


As with the previous functions, you can obtain a prediction using predict_rf():

predict_rf(randomforest,datapoint)
#> [1] 196310


The predictive performance of the random forest can also be measured using a test:

test_rf(randomforest, ames[500:1000, ])

#>          max          min         mean       median 
#> 226.44126074   0.04279835  20.06655144  11.62264789
